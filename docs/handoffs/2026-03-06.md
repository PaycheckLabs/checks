## 5) Explorer UI Status (Mint + Preview flow)

We are implementing Payment Checks first.

Target flow:
1) User connects wallet on checks.xyz landing/portfolio
2) Mint Check button routes to Mint page
3) User fills:
   - Basic info (name, theme, memo)
   - Distribution strategy (Payment)
   - Recipient address (or “Mint to my address” autofill)
   - Conditions (Instant Claim or Post-Dated date)
   - Collateral (MockUSD) and network gate (Amoy)
4) Preview page shows the check details (no serial or QR yet)
5) User clicks Mint Now
6) After mint:
   - Serial and QR are generated and rendered on the final minted check image
   - User is redirected to the serial route `/testnet/<SERIAL>` for explorer view

Current pages:
- `pages/testnet/payment/mint.tsx`
- `pages/testnet/payment/preview.tsx`

Contract constants (testnet):
- `lib/testnetContracts.ts` points to the canonical PaymentChecks + MockUSD deployment.

## 8) Challenges / Lessons

- Small isolated commits keep CI stable. Large restructuring creates avoidable backtracking.
- The best stability strategy is “one brick at a time”:
  - One feature branch
  - One green commit
  - Merge
  - Repeat
